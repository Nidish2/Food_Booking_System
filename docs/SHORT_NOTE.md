# Developer's Note on AI-Assisted Development

During the development of the Hotel Booking System, I leveraged a variety of AI-assisted tools to streamline repetitive tasks and optimize my workflow. I used ChatGPT primarily for debugging minor runtime warnings and solving simple syntax errors. For boilerplate scaffolding—such as defining initial Prisma model definitions and mapping out routing interfaces—I utilized Gemini, which also served as a reliable sounding board to verify my architectural choices and implementation plans. Additionally, Codex was running locally within the editor to provide context-aware autocomplete suggestions, accelerating line-by-line coding.

These tools were highly effective in reducing the time spent on boilerplate configuration, allowing me to focus on the application's core logic. Specifically, they helped scaffold the Prisma schema setup, generate initial unit and integration test configurations in Vitest, and write the extensive seeder files. By automating the setup of test assertions and structure, the AI tools allowed me to allocate more time to refining the database transaction logic and ensuring the user interface remained clean and fully responsive.

The primary challenge encountered during development was coordinating the time-dependent review validation rules within our integration tests. Because the system prevents future bookings from being marked as completed, testing the review endpoint required bypassing standard API routes to directly inject past checkout records into the database. Resolving this required analyzing Prisma transaction behaviors and structuring setup hooks to reset database states correctly. Overcoming these test environment constraints, along with ordering the CORS and rate-limiting middleware to prevent preflight blocks, required close attention to execution lifecycle flows and manual debugging of network preflight logs.
